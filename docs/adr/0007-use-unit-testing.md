# ADR-007: Use unit testing

Date: ~2020
Status: Accepted
Category: testing
Confidence: 80%

## Context

The need for a comprehensive and automated testing framework to ensure system reliability. The presence of testing configurations in tsconfig.json indicates the use of unit testing.

## Decision

Unit testing was chosen as the testing framework due to its comprehensiveness and automation.

## Consequences

### Positive

- Comprehensiveness
- Automation

### Negative

- Limited test coverage

## Alternatives Considered

- **Integration testing**: Not chosen due to increased complexity

## Evidence trail

- `tsconfig.json`
